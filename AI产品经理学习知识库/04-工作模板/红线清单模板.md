# AI 红线清单模板

红线清单用于明确 AI 产品绝不应该做什么。它比功能清单更重要，因为 AI 产品的风险通常来自边界不清。

## 1. 数据红线

- [ ] 不得访问用户无权限的数据。
- [ ] 不得把敏感数据写入可公开日志。
- [ ] 不得把客户隐私用于未授权训练。
- [ ] 不得在未脱敏情况下展示身份证、手机号、邮箱、财务信息等敏感字段。

## 2. 输出红线

- [ ] 不得编造数字、事实、来源或用户原话。
- [ ] 不得在证据不足时给出确定结论。
- [ ] 不得输出违反法律法规或平台政策的内容。
- [ ] 不得输出歧视、骚扰、违法、危险建议。
- [ ] 不得把内部报错或系统提示暴露给外部用户。

## 3. 操作红线

- [ ] 不得在用户未确认时对外发送消息。
- [ ] 不得在用户未确认时提交审批、订单或工单。
- [ ] 不得自动删除、覆盖或修改关键业务数据。
- [ ] 不得绕过人工审核执行高风险动作。

## 4. 知识库红线

- [ ] 不得检索过期知识作为当前答案依据。
- [ ] 不得跨部门或跨租户检索无权限内容。
- [ ] 不得在多个来源冲突时强行输出唯一答案。
- [ ] 找不到资料时不得硬答。

## 5. Agent 红线

- [ ] 不得无限循环执行任务。
- [ ] 不得滥用工具调用。
- [ ] 不得把工具返回的敏感信息无过滤展示给用户。
- [ ] 不得在没有审计日志时执行关键操作。

## 6. 红线触发后的 UX

说明命中红线时如何提示用户。

建议原则：

- 提示要清晰。
- 不暴露内部安全规则细节。
- 给出可继续完成任务的替代路径。
- 高风险场景引导人工处理。
