# AI 风险分级模板

这份模板用于判断 AI 功能的风险等级，并决定相应的控制措施。

## 1. 风险等级

| 等级 | 描述 | 示例 | 控制要求 |
| --- | --- | --- | --- |
| L1 低风险 | 输出仅供参考，不影响关键决策 | 文案改写、会议总结 | 可直接生成，保留反馈入口 |
| L2 中风险 | 影响工作效率或内部判断 | 知识库问答、需求分类 | 需要引用、拒答、日志 |
| L3 高风险 | 影响客户、财务、合同、人事等重要决策 | 合同初审、客户回复建议 | 需要人工确认、审计、红队测试 |
| L4 禁止/极高风险 | 错误后果严重或合规不可接受 | 自动法律建议、自动医疗诊断、无确认转账 | 不上线或仅人工辅助 |

## 2. 风险维度

| 维度 | 问题 | 评分 |
| --- | --- | --- |
| 数据敏感度 | 是否涉及隐私、财务、合同、HR、客户机密 |  |
| 决策影响 | 输出是否影响重要决策 |  |
| 自动化程度 | AI 是否会自动执行操作 |  |
| 可逆性 | 错误后是否可以撤回 |  |
| 用户判断能力 | 用户是否能判断 AI 输出对错 |  |
| 权限复杂度 | 是否涉及跨部门、跨租户、分级权限 |  |
| 合规要求 | 是否涉及法律、医疗、金融、政企等强监管 |  |

## 3. 控制措施

| 风险等级 | 必要措施 |
| --- | --- |
| L1 | 用户反馈、基础日志、基础安全过滤 |
| L2 | 引用来源、拒答/澄清、权限过滤、Eval case |
| L3 | 人工确认、审计日志、红队测试、灰度发布、质量周报 |
| L4 | 原则上不做；如必须做，只允许辅助，不允许自动决策 |

## 4. Skill 暴露策略

| Skill 类型 | 风险等级参考 | 是否允许用户直通 | 必要控制 |
| --- | --- | --- | --- |
| 只读查询 | L1-L2 | 允许 | 权限校验、日志 |
| 生成类 | L1-L2 | 允许 | Schema、禁用词、编辑/采纳 |
| 通知/消息 | L2-L3 | 谨慎允许 | 模板化、预览、确认 |
| 写数据 | L3 | 不允许直通 | Agent/Workflow、二次确认、审计 |
| 内部工具 | L3-L4 | 禁止 | 内部权限、不可外显 |

## 5. 上线审批

| 风险等级 | 审批人 | 上线条件 |
| --- | --- | --- |
| L1 | Product Owner | 基础测试通过 |
| L2 | Product Owner + 业务负责人 | Eval 达标，权限验证通过 |
| L3 | Product Owner + 安全/法务 + 业务负责人 | 红队测试、审计、人工确认机制通过 |
| L4 | 高层专项审批 | 原则上不建议上线 |

## 6. 复审机制

说明：

- 风险等级多久复审一次。
- 模型变更后是否重新评估。
- 业务范围扩大后是否重新评估。
- 出现事故后如何降级或回滚。
