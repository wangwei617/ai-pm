# AI 链路全景图模板

这份模板用于把 AI 功能从用户触发到结果展示的完整链路画清楚，尤其适合在 PRD 首页或评审会上使用。

## 1. 功能名称

填写 AI 功能名称。

## 2. 主链路

```text
用户触发
→ 输入校验
→ Gateway 路由
→ 数据/知识检索
→ Prompt 组装
→ 模型调用
→ 输出解析
→ 质量校验
→ 安全过滤
→ 展示给用户
→ 用户采纳/编辑/反馈
→ 回流 Eval
```

## 5. Gateway 路由决策

| 入口 | 意图特征 | 工具需求 | 风险等级 | 路由结果 | 加载 Harness |
| --- | --- | --- | --- | --- | --- |
| 聊天 |  |  |  | RAG/Safe Skill/Agent/拒绝/人工审核 |  |
| CLI |  |  |  | RAG/Safe Skill/Agent/拒绝/人工审核 |  |
| Webhook/API |  |  |  | RAG/Safe Skill/Agent/拒绝/人工审核 |  |
| 定时任务 |  |  |  | RAG/Safe Skill/Agent/拒绝/人工审核 |  |

原则：

- 路由决策不写死在 Prompt 里。
- LLM 可以辅助意图识别，但最终路由由 Gateway 和策略规则裁决。
- 高风险请求必须进入受限 Harness 或人工审核。

## 9. 迷你示例

场景：用户输入「帮我把目录下所有 CSV 合并并统计空值率」。

链路：

```text
聊天入口
→ Gateway 判断为文件分析任务
→ 路由到 Agent 链路
→ 加载低资源沙箱 Harness
→ Step1 查看目录
→ Step2 读取样本
→ Step3 执行受限脚本
→ Step4 验证产物
→ Step5 返回结果
```
