# Agent 上线验收清单

这份清单用于判断一个 Agent 是否可以进入试点或上线。

## 1. 目标与边界

- [ ] Agent 目标明确。
- [ ] 适用场景明确。
- [ ] 不适用场景明确。
- [ ] 用户启动入口明确。
- [ ] 最终完成条件明确。

## 2. 工具白名单

- [ ] 已列出所有可调用工具。
- [ ] 白名单之外的工具会被系统拦截。
- [ ] 每个工具的输入参数已定义。
- [ ] 每个工具的输出格式已定义。
- [ ] 每个工具的风险等级已定义。
- [ ] 工具参数有校验机制。

## 3. 权限控制

- [ ] Harness 层有权限校验。
- [ ] 工具内部也有权限校验。
- [ ] 用户只能访问自己有权限的数据。
- [ ] 权限不足时有明确提示。
- [ ] 权限失败会写入日志。

## 4. 停止条件

- [ ] 最大步数已设置。
- [ ] 单步超时已设置。
- [ ] 总任务超时已设置。
- [ ] 最大成本已设置。
- [ ] 循环检测已设计。
- [ ] 达到停止条件后不会继续调用工具。

## 5. 写操作确认

- [ ] 发送消息前必须预览。
- [ ] 创建工单前必须确认。
- [ ] 修改数据前必须确认。
- [ ] 提交审批前必须确认。
- [ ] 删除或覆盖操作原则上禁止，或需要更高等级确认。
- [ ] 用户确认记录会进入 Trace。

## 6. 失败处理

- [ ] 工具返回空有处理。
- [ ] 工具返回错误有处理。
- [ ] 工具返回超长内容有处理。
- [ ] 工具返回疑似注入内容有处理。
- [ ] Agent 达到最大步数有提示。
- [ ] 用户可以中断。
- [ ] 必要时可以转人工。

## 7. Trace 与审计

- [ ] 每次任务有 trace_id。
- [ ] 每步工具调用有 step_id。
- [ ] 工具名称、参数、返回、延迟均可追溯。
- [ ] 用户确认记录可追溯。
- [ ] 最终输出可追溯。
- [ ] 事故后可以还原执行过程。

## 8. 评测

- [ ] 有正常 case。
- [ ] 有边界 case。
- [ ] 有红线 case。
- [ ] 有工具失败 case。
- [ ] 有权限失败 case。
- [ ] 有写操作确认 case。
- [ ] 有最大步数/超时 case。

## 9. 上线指标

- [ ] 任务完成率。
- [ ] 工具调用成功率。
- [ ] 人工接管率。
- [ ] 写操作确认率。
- [ ] Trace 完整率。
- [ ] 单次任务成本。
- [ ] p95 完成时长。

## 10. 一票否决项

以下任一项未满足，不建议上线：

- [ ] 没有工具白名单。
- [ ] 没有停止条件。
- [ ] 写操作没有人工确认。
- [ ] 权限只写在 Prompt 里，没有系统级校验。
- [ ] 没有 Trace。
- [ ] 无法回滚或中断高风险操作。
