## 3. 分类规则

### 可直通暴露

满足：

- 低风险。
- 意图确定。
- 参数可校验。
- 无副作用或可撤回。
- 用户能理解结果。

适合入口：

- 按钮。
- 斜杠命令。
- 表单。

### 必须经过 Agent/Workflow

满足任一：

- 需要多步骤判断。
- 需要多个 Skill 串联。
- 有副作用。
- 需要预览和确认。
- 需要根据工具返回继续决策。

适合入口：

- Agent 对话。
- 工作流。
- 预览卡确认。

### 必须锁死不外显

满足任一：

- 内部调试工具。
- 系统日志查询。
- 安全扫描。
- 高权限操作。
- 用户无法理解或无法判断风险。

适合入口：

- 内部 Harness。
- 后台管理。
- 不对普通用户展示。

## 6. 上线验收

- [ ] 每个 Skill 都有 owner。
- [ ] 每个 Skill 都有输入输出 Schema。
- [ ] 每个 Skill 都标注风险等级。
- [ ] 每个 Skill 都标注是否有副作用。

- [ ] 内部工具不对普通用户外显。

- [ ] 所有 Skill 均纳入白名单管理。
