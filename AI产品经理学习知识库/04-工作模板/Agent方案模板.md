## 5. 可用工具 / Skill

工具白名单：

| Skill/工具 | 类型 | 用途 | 输入 | 输出 | 权限要求 | 风险等级 | 允许用户直通 | 是否需确认 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- |
|  | 只读/生成/消息/写数据/内部工具 |  |  |  |  |  | 是/否 | 是/否 |

说明：

- 白名单之外的工具必须被拦截。
- 工具参数必须校验。
- 高风险工具必须有人工确认。
- 低风险、可校验、无副作用的 Skill 可以考虑用户直通。
- 有副作用的 Skill 不允许直通，必须走预览、确认和审计。

## 6. Skill 暴露策略

| Skill 类型 | 是否允许用户直通 | 推荐入口 | 控制措施 |
| --- | --- | --- | --- |
| 只读查询 | 是 | 按钮/指令/表单 | 权限校验、日志 |
| 生成类 | 是 | 生成卡片 | Schema、编辑、采纳、反馈 |
| 通知/消息 | 谨慎 | 预览卡确认 | 模板、预览、确认 |
| 写数据 | 否 | Agent/Workflow 后置 | 二次确认、审计、可撤回 |
| 内部工具 | 否 | 不外显 | 内部权限、审计 |
