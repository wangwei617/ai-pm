# Prompt 设计模板

## 1. Prompt 名称

填写 Prompt 的用途。

例如：用户访谈痛点提取 Prompt。

## 2. 使用场景

说明这个 Prompt 用在哪个产品流程中。

## 3. 输入变量

列出需要传给模型的变量。

例如：

- `{{user_text}}`
- `{{business_context}}`
- `{{output_language}}`

## 4. Prompt 正文

```text
[System Prompt]
你是{{role}}。

任务：
{{task}}

要求：
- {{rule_1}}
- {{rule_2}}
- {{rule_3}}

[Context]
{{context}}

[User Input]
{{input}}

输出格式：
{{output_schema}}
```

## 5. 输出格式

建议使用 JSON 或固定字段。

### Output Schema

| 字段 | 类型 | 约束 | 示例 | 失败处理 |
| --- | --- | --- | --- | --- |
|  |  |  |  |  |

要求说明：

- 是否必须输出裸 JSON。
- 是否禁止 Markdown 代码块。
- 是否允许额外解释文字。
- 字段缺失时如何处理。
- JSON 解析失败时如何处理。

## 6. 示例

提供至少 1 个输入和输出示例。

## 7. 失败处理

说明当输入不足、模型不确定或无法回答时应该如何输出。
