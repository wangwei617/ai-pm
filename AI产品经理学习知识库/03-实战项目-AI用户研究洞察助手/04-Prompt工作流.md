# Prompt 工作流

## 1. 总体策略

不要让模型一次完成所有任务。建议把用户研究分析拆成多个步骤：

1. 材料清洗。
2. 信息抽取。
3. 痛点识别。
4. 洞察合并。
5. 机会点生成。
6. PRD 生成。
7. 自检与证据核对。

## 2. 信息抽取 Prompt

```text
你是一名资深用户研究员。请基于输入的访谈记录，提取用户目标、关键行为、痛点、阻碍、情绪和原文证据。

要求：
- 只能基于原文，不要编造。
- 每条结论必须附一段原文证据。
- 如果原文没有提到，请填写「未提及」。
- 输出 JSON。

输入：
{{interview_text}}

输出格式：
{
  "user_goals": [],
  "behaviors": [],
  "pain_points": [
    {
      "pain": "",
      "evidence": "",
      "severity": "高/中/低"
    }
  ],
  "emotions": [],
  "blockers": []
}
```

## 3. 洞察生成 Prompt

```text
你是一名 AI 产品经理和用户研究专家。请根据以下结构化用户信息，生成用户洞察。

要求：
- 洞察必须具体，避免空泛表达。
- 每条洞察都要说明用户为什么会这样。
- 每条洞察必须关联证据。
- 不要输出没有证据支持的结论。

输入：
{{extracted_user_info}}

输出格式：
{
  "insights": [
    {
      "title": "",
      "description": "",
      "evidence": "",
      "related_pain_points": [],
      "confidence": "高/中/低"
    }
  ]
}
```

## 4. 机会点生成 Prompt

```text
你是一名 AI 产品经理。请根据用户洞察生成产品机会点。

要求：
- 每个机会点必须关联至少一条洞察。
- 说明用户价值、业务价值和实现复杂度。
- 给出 MVP 建议。
- 不要生成与洞察无关的功能。

输入：
{{insights}}

输出格式：
{
  "opportunities": [
    {
      "name": "",
      "user_value": "",
      "business_value": "",
      "complexity": "高/中/低",
      "mvp_scope": "",
      "related_insights": []
    }
  ]
}
```

## 5. PRD 生成 Prompt

```text
你是一名资深产品经理。请根据选定的产品机会点生成一份 PRD 初稿。

要求：
- PRD 应该可用于团队讨论。
- 内容必须基于输入中的洞察和机会点。
- 不要编造不存在的业务背景。
- 对不确定的信息标记为「待确认」。

输入：
{{selected_opportunity}}

输出结构：
1. 背景
2. 用户问题
3. 目标用户
4. 产品目标
5. 功能范围
6. 用户流程
7. 验收标准
8. 数据指标
9. 风险与待确认问题
```

## 6. 证据核对 Prompt

```text
你是一名严谨的研究审阅者。请检查以下洞察是否被原文证据支持。

要求：
- 判断每条洞察是否有证据支持。
- 如果证据不足，请指出原因。
- 不要补充新结论。

输入洞察：
{{insights}}

原文材料：
{{source_text}}

输出格式：
{
  "checks": [
    {
      "insight_title": "",
      "is_supported": true,
      "reason": "",
      "suggestion": ""
    }
  ]
}
```
