# API 与技术协作基础

## 1. 这个知识点是什么

API 是不同系统之间交换数据和调用能力的接口。

AI PM 不需要写复杂代码，但要理解 API 的基本逻辑，这样才能和工程师沟通 AI 能力如何接入产品。

## 2. 为什么重要

很多 AI 产品不是独立 Demo，而是要嵌入真实系统。

例如：

- 从 CRM 读取客户信息。
- 从知识库检索资料。
- 调用模型生成回复。
- 把结果写回工单系统。
- 发送通知。

这些都依赖 API。

## 3. 基础概念

### Request

请求，是客户端发给服务端的数据。

例如用户提交一段访谈文本，请求 AI 分析。

### Response

响应，是服务端返回给客户端的数据。

例如 AI 返回洞察列表和证据引用。

### JSON

JSON 是常见的数据格式。

AI 产品中经常要求模型输出 JSON，因为系统可以稳定读取字段。

### Endpoint

Endpoint 是接口地址。

例如 `/api/analyze-interview`。

### Authentication

鉴权，用于确认调用者是否有权限。

AI 产品尤其要注意权限，因为模型可能接触敏感数据。

### Rate Limit

速率限制，用于控制接口调用频率。

如果调用太频繁，可能被限制或成本飙升。

## 4. AI 产品中的典型接口

- 提交 AI 任务接口。
- 查询任务状态接口。
- 获取 AI 结果接口。
- 用户反馈接口。
- 知识库检索接口。
- 文件上传接口。
- 工具调用接口。
- 成本统计接口。

## 5. 工作中怎么用

写 AI PRD 时，你不需要写完整技术方案，但要说明：

- 前端需要传什么数据。
- 后端需要返回什么结果。
- AI 输出字段是什么。
- 是否需要异步任务。
- 是否需要任务状态。
- 是否需要错误码。
- 是否需要权限校验。

## 6. 可交付物

- 输入输出字段说明。
- API 需求说明。
- 状态流转图。
- 错误码说明。
- 权限说明。

## 7. 常见坑

- 只描述页面，不描述数据流。
- 没有定义 AI 输出字段。
- 忽略长任务的等待和查询状态。
- 没有考虑接口失败。
- 没有考虑权限和审计。
